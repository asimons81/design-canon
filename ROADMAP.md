# Roadmap

## v0.1: The Canon Compiles

- [x] Atomic JSON rule catalog
- [x] Marketing, product-app, and editorial profiles
- [x] `DESIGN.md`, `SKILL.md`, and agent-instruction compilation
- [x] Dependency-free static linter
- [x] Node test suite and deliberately bad fixture
- [ ] Rule suppression format with required rationale
- [ ] CI workflow and release packaging
- [ ] Codex, Hermes Agent, Claude Code, and Cursor install guides

## v0.2: The Canon Can See

- Screenshot capture adapter
- Structured visual-review JSON contract
- Dual-model review option
- Before/after comparison report
- Viewport and interaction-state matrix
- Accessibility checks backed by browser evidence

## v0.3: The Canon Learns Taste

- Project-local approvals and rejections
- Preference confidence, conflict detection, and time decay
- Style extraction from approved screenshots and existing code
- Privacy-preserving local storage and portable export

## v0.4: Prove It Works

- Fixed benchmark tasks and golden briefs
- Paired runs with and without compiled canons
- Human-blind comparison workflow
- Token, latency, pass-rate, accessibility, and visual-quality metrics
- Public leaderboard with reproducible run manifests

## v1.0: Open Design Policy Runtime

- Stable rule and profile schemas
- Plugin registry with signature and provenance metadata
- Policy composition and semantic-version migration
- IDE and CI integrations
- Governance process for controversial or subjective rules
